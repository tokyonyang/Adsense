# main.py 텔레그램 리포트 정리 패치

## 확인된 원인

현재 `main.py` 안에서 텔레그램 메시지를 직접 만들고 있습니다.

수정 대상 함수:

- `_news_links_html()`
- `_daily_digest_to_telegram_text()`

기존 코드에서는 아래 정보가 텔레그램에 그대로 표시되었습니다.

- 관심도
- Google 조회수
- 근거강도
- 네이버 신호
- DataLab
- 수집경로
- 작성각도
- 자동 대체 검색 로그
- 링크1, 링크2, 링크3 라벨

## 적용 후 변경

핫이슈는 아래처럼 표시됩니다.

```text
🔥 오늘의 핫이슈 TOP 10

1. [경제·금융] 정부, 하반기 전기·가스요금 동결…
근거자료:
  1) 기사 제목 (언론사 · 시간)
  2) 기사 제목 (언론사 · 시간)
```

실제 URL이 있는 경우에는 `기사 제목` 자체가 텔레그램 클릭 링크가 됩니다.
URL이 없는 경우에는 링크 문구를 표시하지 않습니다.

## 적용 방법

1. GitHub 레포에서 기존 `main.py` 파일을 엽니다.
2. 이 패키지의 `main.py` 내용으로 전체 교체합니다.
3. Commit changes 합니다.
4. GitHub Actions에서 다시 Run workflow를 실행합니다.

## 주의

`telegram_notify.py`가 아니라 `main.py`에서 메시지 포맷을 직접 생성하고 있었으므로,
이번 패치는 `main.py` 교체가 핵심입니다.
